\begin{problem}{TTS Rights}{tts.in}{tts.out}{5 seconds}{64 Megabytes}

TTS (Trivial Testing System) используется в СПбГУ для того, чтобы
студенты могли решать контесты из дома. TTS использует следующую
систему прав. Каждый объект (команда, задача, контест, условие, и т.д.) 
должен состоять ровно в одной группе. Группы организованы во взвешенный 
ориентированный граф с целыми весами рёбер. Права объекта $a$ на объект 
$b$ --- это права группы, в которой состоит $a$, на группу, в которой состоит
$b$. Права группы $A$ на группу $B$ могут быть получены следующим способом.
Система рассматривает все непустые пути из $A$ в $B$. Каждому пути соответствует
число, которое можно вычислить как логическое \texttt{И} весов всех рёбер пути.
Затем система вычисляет число $R$ --- логическое \texttt{ИЛИ} чисел, 
соответствующих путям из $A$ в $B$. Это число определяет права $A$ на $B$.
Четыре младших бита --- это права приоритета 0. Они располагаются так:
бит 0 это \texttt{X} (права на запуск), бит 1 --- \texttt{W} (права на запись),
бит 2 --- \texttt{R} (права на чтение), бит 3 --- \texttt{A} (права на администрирование).
Тем же образом биты 4--7 определяют права приоритета 1, 8--11 --- приоритета 2, и так далее. 
Максимальный приоритет --- седьмой. Окончательные же права определяются так: система делает цикл 
по всем приоритетам от 0-го до 7-го. При этом для чётных приоритетов система устанавливает включённые биты,
а для нечётных --- обнуляет включённые биты. Полученные 4 бита и есть окончательные права.

В графе есть две вершины, \texttt{initial} и \texttt{final}.
Из вершины \texttt{initial} идут рёбра веса $-1$ во все остальные вершины.
В вершину \texttt{final} входят рёбра веса $-1$ изо всех остальных вершин.
Невозможно поменять рёбра \textbf{из} \texttt{initial} и 
\textbf{в} \texttt{final}.

Ваша задача --- написать программу, которая поддерживает эффективное
вычисление прав.

%Your task is to write a program which supports effective rights evaluation

\InputFile

Все данные в TTS содержатся в так называемом \texttt{логе}. Лог это
файл из текстовых строк, каждая из которых содержит 4 или больше полей,
разделённых символами табуляции (код ASCII 9). Первое поле --- это всегда
UNIX-время создания соответствующей записи. Второе поле определяет собственно
запись. В третьем содержится описание компьютера, с которого сделана запись.
Четвёртое --- это идентификатор записи. Остальные поля, если они есть --- это
параметры записи.

%All data in TTS are contained in the so-called \textit{log}. The log is
%a file of text strings, each of which contains four or more fields,
%separated by tabulation characters (ASCII code 9). The first field
%is always equal to UNIX time of creation of the corresponding record.
%The second describes the record itself. The third field describes the computer
%which created the record. The fourth one is an identifier of the record.
%The other optional fields are parameters of the record.

В этой задаче лог будет содержать только записи о присвоении прав и о 
вычислении прав. Синтаксис записей следующий:

%In this task your log will contain only group rights assignment records
%and rights evaluation operations. The syntax is as follows:

\def\z#1{$\langle$\texttt{#1}$\rangle$}

\texttt{
\begin{tabular}{l l l l l l} 
\z{utime}\quad&(+$|$-$|$=)GR\quad&\z{ip}\quad&\z{src-node}\quad&\z{rights-mask}\quad&\{\z{dest-nodes}\}\cr
\z{utime}\quad&(+$|$-$|$=)RG\quad&\z{ip}\quad&\z{dest-node}\quad&\z{rights-mask}\quad &\{\z{src-nodes}\}\cr
\z{utime}\quad&RIGHTS\quad&\z{ip}\quad&\z{src-node}\quad&\z{dest-node}
\end{tabular}
}

Эти команды действуют на все рёбра из \z{src-node} в \z{dest-nodes}
или на все рёбра из \z{src-nodes} в \z{dest-node}.
Операции с префиксом `\texttt{+}' добавляют указанные 
права к ребру (то есть выполняет операцию логического \texttt{ИЛИ}),
с префиксом `\texttt{-}' убирают права (то есть обнуляют все биты,
которые содержит \z{rights-mask}), а с префиксом `\texttt{=}' заменяют
числа на соответствующих рёбрах.

%These commands operate on all edges from \z{src-node} to \z{dest-nodes} or for edges
%from \z{src-nodes} to \z{dest-node}. Operations with `\texttt{+}' prefix add specified
%rights to an edge (performing a logical \texttt{or} operation), 
%`\texttt{-}' operations revoke rights (clearing all bits which are set in \z{rights-mask})
%and `\texttt{=}' operations replace the numbers on the corresponding edges.

Если маска указана как `\texttt{-}', то соответствующее число
считается равным $-1$.

%If the mask is specified as `\texttt{-}', the corresponding number
%is considered to be equal to $-1$.

\z{rights-mask} --- это десятичное или шестнадцатеричное число. В последнем случае 
число начинается с префикса \texttt{0x}. Числа рассматриваются по модулю $2^{32}$
и не могут по абсолютной величине превосходит $2^{32}-1$. Таким образом, каждое
число, отличное от 0, имеет два различных представления, отличающихся на $2^{32}$.
Например, $-1=\mathtt{0xffffffff}=4294967295$. Знак минус может находиться только 
перед числом, например \texttt{-0x25}.

%\z{rights-mask} is a decimal or hexadecimal number. In the latter case the number
%%is prefixed by \texttt{0x}. The numbers are considered modulo $2^{32}$ and could
%not have absolute value more than $2^{32}-1$. So each number other than zero
%has two possible representations differing by $2^{32}$, for example, 
%$-1=\hbox{0xffffffff}=4294967295$. Minus sign may only appear before numbers,
%for example: \texttt{-0x25}. 

\OutputFile

Для каждой команды \texttt{RIGHTS} из входного файла, выведите сообщения 
о правах \z{src-node} на \z{dest-node}. Следуйте формату вывода в примере.
Права должны выводиться от старших битов к младшим. Размер лога будет
не больше 64 килобайт.

%For each \texttt{RIGHTS} command in input, output a message about effective
