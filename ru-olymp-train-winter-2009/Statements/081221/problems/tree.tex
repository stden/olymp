\begin{problem}{Дерево}{\textsl{нет}}{\textsl{нет}}
{2 секунды}{64 мегабайта}

Есть дерево из $n$ вершин. Степень каждой вершины не превосходит $5$.
Алиса и Боб играют в такую игру: Алиса загадывает какую-то вершину,
Боб пытается её угадать.
В процессе игры Боб может указать на ребро, и Алиса сообщает, по какую
сторону от ребра лежит загаданная вершина.
Вам предлагается написать программу, которая играет за Боба
и угадывает загаданную вершину не более чем за $100$ запросов.

\Details

Ваша программа должна взаимодействовать с модулем. Для этого она должна
подключить модуль \t{treeunit} на Pascal или заголовочный файл
\t{treeunit.h} на C/C++. Необходимые материалы вы можете найти
на сайте сборов.

В модуле доступны следующие функции:

\begin{verbatim}
procedure init;

void init();
\end{verbatim}

Эта функция служебная. Вы должны вызвать её в начале вашей программы.

\begin{verbatim}
function getN: longint;

int getN(void);
\end{verbatim}

Функция возвращает $n$ --- количество вершин в дереве
($1 \leqslant n \leqslant 200\,000$).

\begin{verbatim}
function getA(edgeNum: longint): longint;
function getB(edgeNum: longint): longint;

int getA(int edgeNum);
int getB(int edgeNum);
\end{verbatim}

Эти функции возвращают номера концов ребра с номером $\mathtt{edgeNum}$
($1 \leqslant \mathtt{edgeNum} \leqslant n - 1$).
Возвращаемые значения~--- натуральные числа от $1$ до $n$.

\begin{verbatim}
function query(edgeNum: longint): longint;

int query(int edgeNum);
\end{verbatim}

С помощью этой функции вы можете узнать по какую сторону
ребра с номером \t{edgeNum} лежит загаданная вершина.
Функция возвращает $0$, если вершина лежит по ту же сторону,
что и \t{getA(edgeNum)}, и $1$ в противном случае.

\begin{verbatim}
procedure report(vertexNum: longint);

void report(int vertexNum);
\end{verbatim}

Вы должны использовать эту функцию для того, чтобы сообщить
ваше предположение о загаданной вершине. Если вы вызовете эту функцию,
то ваша программа будет завершена.

Если будет вызвана какая-либо функция с некорректными параметрами,
либо не будет вызвана \t{init} в начале программы, то ваша программа
также будет завершена.

Для отладки ваших решений на сайте сборов можно будет скачать демо-версии
модулей. Эти версии читают данные из файла \t{input.txt} и записывают
результат в стандартный поток вывода. Их интерфейс взаимодействия с вашей
программой будет идентичен используемому при тестировании.

При компиляции программ на C++ используйте следующую команду:

\t{g++ -o a a.cpp treeunit.o}

При компиляции программ на C используйте следующую команду:

\t{gcc -o a a.c treeunit.o}

При компиляции программ на Pascal используйте следующую команду:

\t{fpc a.pas}

\InputFile

В первой строке находится $n$~--- количество вершин в дереве. Следующие
$n - 1$ строк содержат описания рёбер дерева~--- каждое ребро задано
номерами своих концов. Следующая строка содержит номер загаданной вершины.

\OutputFile

В стандартном потоке вывода будет содержаться некоторая полезная
информация~--- лог взаимодействия вашей
программы с модулем и описание произошедших ошибок.

\Example

\renewcommand{\InputFileName}{input.txt}
\renewcommand{\OutputFileName}{стандартный поток вывода}
\begin{example}
\exmp{
5
1 2
2 3
3 4
4 5
3
}{
init()
getN()
getA(1)
getB(1)
query(1)
getA(2)
getB(2)
query(2)
getA(3)
getB(3)
query(3)
getA(4)
getB(4)
query(4)
report(3)
ok
}%
\end{example}

\end{problem}
